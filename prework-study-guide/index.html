HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • Parents elements holds Child elements.
  • HTML elements are case-sensitive.
  • Declarations are not case-sensitive.
  • URL is the abbreviation for Uniform Resource Locator.
  • HTML (Hypertext Markup Language) is used to give the content structure.

CSS

  • CSS is an abbreviation for Cascading Style Sheets.
  • There are three ways to style a webpage using CSS: inline CSS, internal CSS style sheet, and external CSS style sheet.
  • Sometimes, we want to apply a rule to all the elements visible on the page. To do that, we use a special wild card symbol, or *
  • The block value assigns each image to start on a new line.
  • `link` element creates the connection to the external file.
  • The rel attribute specifies the relationship between the current document and the linked document resource.
  • The href attribute specifies the location (URL) of the external resource.
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.

GIT

  • git config --global user.name "JohnDoe": Set your Git username.
  • git config --global user.name: Shows your username.
  • git config --global user.email "JohnDoe@gmail.com": Set your Git e-mail.
  • git remote -v: To verify if has connection between the local version of the repo and remote repo.
  • git status: Checks what branch we are currently on.
  • git checkout -b branch-name: Creates a new branch and switches to it.
  • git add -A: To commit all files in the working branch.
  • git commit -m "commit title example": To commit the changes to the repository
    (We use the -m flag to associate a message with our commit.)
  • git pull origin main: To pull in the base branch
  • git push origin branch-name: To push up all of the changes

JavaScript

  • JS is an abbreviation for JavaScript
  • JavaScript is a high-level dynamic programming language that adds interactivity to your website. It was invented by Brendan Eich.
  • APIs is an abbreviation for Application Programming Interfaces
  • The script tag (< script src="scripts/main.js">< /script>) needs to be before the closing body tag
  • A semicolon at the end of a line indicates where a statement ends.
  • JavaScript is case sensitive.
  • Variables are containers that store values/data in our code. You start by declaring a variable with the `var` keyword, followed by the name you give to the variable.
  • After declaring a variable, you can give it a value:
    myVariable = "Bob"; or let myVariable = "Bob";
  • You retrieve the value by calling the variable name:
    myVariable;
  • After assigning a value to a variable, you can change it later in the code:
    let myVariable = "Bob";
    myVariable = "Steve";
  • JS data types:

    -String is a sequence of text known as a string. To signify that the value is a string, enclose it in single or double quote marks.
    let myVariable = 'Bob'; or
    let myVariable = "Bob";

    -Number. Numbers don't have quotes around them.

    -Boolean is a True/False value. The words true and false are special keywords that don't need quote marks.

    -Array is a structure that allows you to store multiple values in a single reference.
    let myVariable = [1,'Bob','Steve',10];
    Refer to each member of the array like this:
    myVariable[0], myVariable[1], etc.

    -Object can be anything. Everything in JavaScript is an object and can be stored in a variable.
    let myVariable = document.querySelector('h1');
    All of the above examples too.
  • You can write comments in JavaScript just as you can in CSS:
    /*
    Everything in between is a comment.
    */
    If your comment contains no line breaks, it's an option to put it behind two slashes like this:
    // This is a comment
  • An operator is a mathematical symbol that produces a result based on two values (or variables).

    -Addition (+): Add two numbers together or combine two strings.

    -Subtraction, Multiplication, Division (-, *, /): These do what you'd expect them to do in basic math.

    -Assignment (=): Assigns a value to a variable.

    -Strict equality (===): This performs a test to see if two values are equal. It returns a true/false (Boolean) result.
    let myVariable = 3;
    myVariable === 4;

    -Not, Does-not-equal (!, !==): This returns the logically opposite value of what it precedes. It turns a true into a false, etc.. When it is used alongside the Equality operator, the negation operator tests whether two values are not equal.
    For "Not", the basic expression is true, but the comparison returns false because we negate it:
    let myVariable = 3;
    !(myVariable === 3);

    "Does-not-equal" gives basically the same result with different syntax. Here we are testing "is myVariable NOT equal to 3". This returns false because myVariable IS equal to 3:

    let myVariable = 3;
    myVariable !== 3;
  • We can use the console.log method to output a message to the web console by adding an argument.
  • Control flow is the order in which a computer executes code in a script. In JavaScript, code runs from the first statement at the top of the page, down to the last statement at the bottom of the page.
  • Control flow can be interrupted, however. We can use conditional statements and loops to control the order in which code is executed, or if it is even executed at all.
  • A value is considered falsy if it is one of the following values: 0, -0, On, "", null, undefined, NaN, or a Boolean data type with the value of false. All other values, including a Boolean with the value of true, are considered truthy.
